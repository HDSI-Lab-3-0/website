# AI Prompt: Project Markdown (projects collection)

## Instructions for AI

Generate a complete markdown document for a **project** in this repo. Output the **entire file** in one fenced code block so it can be copied into `src/content/projects/<slug>.md` (or `.mdx`).

Match `src/content.config.ts` (`projects` collection). Frontmatter is validated with Zod; image fields use Astro `image()` — paths must be **relative to the project file** into `src/assets/`.

---

## Frontmatter

### Required

| Field | Type | Notes |
| --- | --- | --- |
| `title` | string | Project title |
| `description` | string | Short blurb for cards and SEO |
| `pubDate` | date | `YYYY-MM-DD` |

### Optional

| Field | Type | Notes |
| --- | --- | --- |
| `updatedDate` | date | `YYYY-MM-DD` |
| `heroImage` | image path | Static hero still (WebP/PNG/JPEG under `src/assets/`) |
| `imageGif` | image path | If set with `heroImage`, **used first** for grid, modal, and detail hero |
| `links` | object | `lessonPlan`, `materials` — optional URL strings |
| `relevance` | string | “Why this matters” callout on the detail page |
| `tags` | string[] | Filters on `/projects`; align with `src/data/tags.json` when possible |
| `estimated_time` | number | Minutes (shown on cards/meta) |
| `sponsor` | string | Comma-separated ok |
| `status` | string | e.g. prototype / active |
| `audience` | string | |
| `engagementType` | string | |
| `location` | string | |

**Images:** put files under e.g. `src/assets/projects/` and reference from frontmatter like `../../assets/projects/my-hero.webp`. Do **not** use `public/` paths for `heroImage` / `imageGif` — those fields go through the asset pipeline.

## Body structure

After frontmatter, use normal Markdown:

- `#` or `##` headings; the layout may dedupe a body `h1` that duplicates `title`.
- Paragraphs, lists, tables, code fences, blockquotes as needed.
- Inline images in the body can use relative paths or imports per Astro/MDX conventions.
